## 2. API 規格

### 2.1 Debug Header 說明

| Header | 說明 | 範例 |
|--------|------|------|
| `X-Trace-ID` | 追蹤 ID，格式: `P{專案}-{維度}-{功能}-{YYYYMMDD}-{序號}` | `P001-API-contacts-20260405-000001` |
| `X-Testing-Case` | 測試案例名稱（供 QA 使用） | `TC-001-list-contacts` |
| `X-QA-Debug-Info` | QA 除錯資訊（JSON 格式） | `{"tester":"QA01","env":"SIT"}` |

### 2.2 API Endpoints

#### GET /api/contacts
- **說明**: 取得所有聯絡人列表
- **Request Headers**: Debug Headers可選
- **Response**: 
  ```json
  {
    "status": "success",
    "data": [
      {"id": "uuid", "name": "string", "email": "string", "phone": "string", "created_at": "ISO8601"}
    ],
    "count": 1
  }
  ```

#### GET /api/contacts/<id>
- **說明**: 取得特定聯絡人
- **Path Parameters**: `id` (required, UUID)
- **Response**: 
  ```json
  {
    "status": "success",
    "data": {"id": "uuid", "name": "string", "email": "string", "phone": "string", "created_at": "ISO8601"}
  }
  ```
- **Error 404**: 
  ```json
  {"status": "error", "message": "Contact not found"}
  ```

#### POST /api/contacts
- **說明**: 新增聯絡人
- **Request Body**:
  ```json
  {
    "name": "string (required)",
    "email": "string (required)",
    "phone": "string (optional)"
  }
  ```
- **Validation**: name, email 必填
- **Response 201**:
  ```json
  {"status": "success", "data": {...}, "message": "Contact created"}
  ```

#### PUT /api/contacts/<id>
- **說明**: 更新聯絡人
- **Request Body**:
  ```json
  {
    "name": "string",
    "email": "string", 
    "phone": "string"
  }
  ```
- **Response 200**:
  ```json
  {"status": "success", "data": {...}, "message": "Contact updated"}
  ```

#### DELETE /api/contacts/<id>
- **說明**: 刪除聯絡人
- **Response 200**:
  ```json
  {"status": "success", "message": "Contact deleted"}
  ```
- **Error 404**: 
  ```json
  {"status": "error", "message": "Contact not found"}
  ```

#### GET /api/contacts/search
- **說明**: 搜尋聯絡人
- **Query Parameters**: `q` (search keyword)
- **Response**:
  ```json
  {
    "status": "success",
    "data": [...],
    "count": 2
  }
  ```

## 3. Trace ID 格式

### 格式
```
P{專案代號}-{維度}-{功能}-{YYYYMMDD}-{序號(6位)}
```

### 範例
- `P001-API-contacts-20260405-000001`
- `P001-WEB-list-20260405-000002`
- `P001-QA-test-20260405-000003`

### 各欄位說明
| 欄位 | 說明 | 範例 |
|------|------|------|
| 專案代號 | P001 (P001-nas-remote) | P001 |
| 維度 | API / WEB / QA / SYS | API |
| 功能 | contacts / list / search / test | contacts |
| 日期 | 8位日期 | 20260405 |
| 序號 | 6位遞增序號 | 000001 |

## 4. Debug Header 規格

### Request Header（接收）
```python
trace_id = request.headers.get('X-Trace-ID', '')        # 追蹤ID
testing_case = request.headers.get('X-Testing-Case', '') # 測試案例
qa_debug_info = request.headers.get('X-QA-Debug-Info', '{}')  # QA除錯資訊
```

### Response Header（回傳）
```python
resp.headers['X-Trace-ID'] = trace_id
resp.headers['X-Testing-Case'] = testing_case
resp.headers['X-QA-Debug-Info'] = qa_debug_info
```

### 自動產生
- 若 Request 未攜帶 `X-Trace-ID`，系統自動產生並回傳
- 序號採用檔案計數方式（traces/counter.txt）
