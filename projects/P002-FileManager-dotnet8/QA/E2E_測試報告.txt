【問題分析】

問題 1: AmbiguousMatchException
原因: 有多個端點匹配 "/" 路徑
可能原因:
  - Components/Routes.razor 定義了 @page "/"
  - Components/Pages/Index.razor 也定義了 @page "/"
  - 造成路由重複

問題 2: 405 Method Not Allowed
原因: GET 請求被拒絕，但 Allow header 顯示 GET 是允許的
這可能是路由匹配失敗後的錯誤處理問題

【修復建議】

1. 檢查 Routes.razor 和 Index.razor 的路由配置
   - 移除重複的 @page "/" 定義

2. 確保 Blazor Server 正確配置:
   ```csharp
   builder.Services.AddRazorComponents()
       .AddInteractiveServerComponents();
   ```

【結論】
前端 Blazor SSR 層級有嚴重問題，需要修复路由衝突和確保正確的 render mode 配置。
