## 測試 1: 路徑遍歷攻擊
- **攻擊目標:** /etc/passwd
- **API:** GET /api/files?path=/etc/passwd
- **預期行為:** 阻擋並回傳錯誤
- **結果:** ✅ 安全性正常

### 回應
```json
{
  "success": false,
  "data": null,
  "message": "Path is outside the root folder.",
  "errorCode": "ACCESS_DENIED"
}
```

### 分析
- 系統正確阻擋了路徑遍歷攻擊
- errorCode: ACCESS_DENIED 明確標示為存取拒絕
- 路徑驗證邏輯正常運作

---

## 測試 2: 不存在路徑
- **測試路徑:** /home/devpro/data/nonexistent_xyz123
- **API:** GET /api/files?path=/home/devpro/data/nonexistent_xyz123
- **預期行為:** 回傳找不到資源
- **結果:** ✅ 正常處理

### 回應
```json
{
  "success": false,
  "data": null,
  "message": "Directory not found: /home/devpro/data/nonexistent_xyz123",
  "errorCode": "PATH_NOT_FOUND"
}
```

### 分析
- 錯誤訊息清晰指出路徑不存在
- 不會洩漏系統敏感資訊
- 適當的 HTTP 狀態碼和錯誤碼
