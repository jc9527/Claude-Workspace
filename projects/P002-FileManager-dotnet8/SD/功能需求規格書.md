# FileManager .NET 8 重建 - 功能需求規格書

**版本：** v1.0
**日期：** 2026-04-05

**基於：** devpro-tw/FileManager (.NET Framework 4.7 + DevExpress)

## 1. 系統概觀

### 1.1 目的

重建一個跨平台的檔案管理系統，可在 Ubuntu（Linux）上運行，使用 .NET 8 + Blazor 技術。

### 1.2 來源系統

| 項目 | 現況 |
|------|------|
| 原始專案 | devpro-tw/FileManager |
| Framework | .NET Framework 4.7 |
| UI 框架 | DevExpress MVC v18.2（Windows only）|
| 儲存位置 | `C:\D\temp`（Windows 路徑）|

## 2. 功能需求

### 2.1 核心功能矩陣

| 編號 | 功能 | 說明 | 優先順序 | 對應 DevExpress |
|------|------|------|----------|-----------------|
| FM-001 | 資料夾瀏覽 | 瀏覽資料夾結構與檔案列表 | ✅ 必要 | FileManager BindToFolder |
| FM-002 | 資料夾建立 | 建立新資料夾 | ✅ 必要 | SettingsEditing.AllowCreate |
| FM-003 | 檔案重新命名 | 重新命名檔案或資料夾 | ✅ 必要 | SettingsEditing.AllowRename |
| FM-004 | 檔案移動 | 移動檔案或資料夾到其他位置 | ✅ 必要 | SettingsEditing.AllowMove |
| FM-005 | 檔案刪除 | 刪除檔案或資料夾 | ✅ 必要 | SettingsEditing.AllowDelete |
| FM-006 | 單一檔案上傳 | 上傳單一檔案 | ✅ 必要 | SettingsUpload.Enabled |
| FM-007 | 多重檔案上傳 | 一次上傳多個檔案 | 🟡 建議 | EnableUploadMultiSelect |
| FM-008 | 拖拉上傳 | 拖曳檔案到頁面上傳 | 🟡 建議 | UseAdvancedUploadMode |
| FM-009 | 檔案下載 | 下載單一檔案 | ✅ 必要 | DownloadRouteValues |
| FM-010 | 多重下載 | 一次下載多個檔案 | 🟡 建議 | EnableMultiSelect |
| FM-011 | 搜尋/篩選 | 依名稱或副檔名篩選 | ✅ 必要 | ShowFilterBox |

## 3. 設定功能（管理後台）

### 3.1 作業權限設定

| 編號 | 設定 | 說明 | 預設值 | DevExpress 對應 |
|------|------|------|--------|-----------------|
| FS-001 | AllowDownload | 允許下載 | true | AllowDownload |
| FS-002 | AllowCreate | 允許建立資料夾 | true | AllowCreate |
| FS-003 | AllowRename | 允許重新命名 | true | AllowRename |
| FS-004 | AllowMove | 允許移動 | true | AllowMove |
| FS-005 | AllowDelete | 允許刪除 | true | AllowDelete |
| FS-006 | EnabledUpload | 允許上傳 | true | EnabledUpload |
| FS-007 | EnableUploadMultiSelect | 允許多選上傳 | true | EnableMultiSelect |
| FS-008 | EnableDownloadMultiSelect | 允許多選下載 | true | EnableMultiSelect |

### 3.2 介面顯示設定

| 編號 | 設定 | 說明 | 預設值 |
|------|------|------|--------|
| FI-001 | ShowPath | 顯示路徑 | true |
| FI-002 | ShowFilterBox | 顯示篩選框 | true |
| FI-003 | ShowExpandButtons | 顯示展開按鈕 | true |
| FI-004 | ShowFolderIcons | 顯示資料夾圖示 | true |
| FI-005 | ShowTreeLines | 顯示樹狀線 | true |
| FI-006 | ShowFolders | 顯示子資料夾 | true |
| FI-007 | ShowParentFolder | 顯示上層資料夾 | true |
| FI-008 | BreadcrumbsVisible | 顯示麵包屑 | true |
| FI-009 | BreadcrumbsPosition | 麵包屑位置（0=上,1=下）| 0 |
| FI-010 | FileListViewMode | 檢視模式（0=縮圖,1=清單）| 1 |

### 3.3 檔案限制設定

| 編號 | 設定 | 說明 | 預設值 |
|------|------|------|--------|
| FL-001 | RootFolder | 根目錄路徑 | /app/data |
| FL-002 | AllowedFileExtensions | 允許的副檔名（*為全部）| * |
| FL-003 | MaxRequestLength | 最大上傳大小（MB）| 4096 |

## 4. 非功能性需求

### 4.1 技術需求

| 項目 | 需求 |
|------|------|
| Framework | .NET 8 |
| UI 框架 | Blazor Server 或 Blazor WebAssembly |
| 資料庫 | 設定檔（JSON 或 appsettings.json）|
| 跨平台 | 支援 Linux（Ubuntu 24.04）|
| 容器化 | 支援 Docker |

### 4.2 效能需求

| 項目 | 目標 |
|------|------|
| 回應時間 | < 500ms（一般操作）|
| 上傳大檔案 | 支援至 4GB |
| 同時上傳 | 支援多檔案上傳 |

### 4.3 安全需求

| 項目 | 需求 |
|------|------|
| 路徑遍歷防護 | 防止 ../ 攻擊 |
| 檔案類型驗證 | 依副檔名過濾 |
| 作業權限控制 | 可設定開/關 |

## 5. 移除 DevExpress 的對應實作

| DevExpress 功能 | Blazor 替代方案 |
|-----------------|-----------------|
| FileManager 元件 | Radzen Blazor / AntDesign Blazor / 自定義 |
| FileManagerSettings | Blazor Component Parameters |
| Download Route | ASP.NET Core Static Files |
| ThumbnailFolder | 自定義縮圖服務 |
| RichEdit Core | 移除（不需要的功能）|

---

## 6. 建議的 UI 框架選擇

| 框架 | 優點 | 缺點 | 適合場景 |
|------|------|------|----------|
| **Radzen Blazor** | 免費、有 FileManager | 功能較基礎 | 中小專案 ✅ |
| **AntDesign Blazor** | UI 漂亮 | 需要學習 | 需要美觀介面 |
| **MudBlazor** | Material Design | 元件數量中等 | 中等複雜度 |
| **自定義** | 完全控制 | 工作量大 | 需要特殊功能 |

## 8. 開發優先順序

### Phase 1（MVP）
1. 資料夾瀏覽（FM-001）
2. 檔案下載（FM-009）
3. 設定讀取（FS-*, FI-*）

### Phase 2（完整功能）
4. 檔案上傳（FM-006, FM-007）
5. 資料夾作業（FM-002, FM-003, FM-004, FM-005）
6. 搜尋/篩選（FM-011）

### Phase 3（增強）
7. 縮圖模式（FV-001）
8. 欄位排序/篩選（FV-003, FV-004）
9. 拖拉上傳（FM-008）

## 10. 風險與對策

| 風險 | 等級 | 對策 |
|------|------|------|
| Blazor 元件庫不支援檔案管理 | 🟡 中 | 使用多個元件組合或自定義 |
| 大檔案上傳效能 | 🟡 中 | 使用流式上傳 |
| 中文路徑支援 | 🟡 中 | 測試並使用 URL 編碼 |
| 遷移複雜度 | 🟢 低 | 採 Greenfield 方式逐步實作 |
