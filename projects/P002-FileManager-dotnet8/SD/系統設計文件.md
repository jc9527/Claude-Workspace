## 1. Blazor 模式選擇

### 1.1 候選模式比較

| 維度 | Blazor Server | Blazor WebAssembly | Blazor United (Auto) |
|------|--------------|-------------------|---------------------|
| 初始載入大小 | ~1-2 MB (HTML+CSS+JS) | ~5-10 MB (+ .NET WASM runtime) | 兩者兼具 |
| 需要 .NET Runtime | ✅ 僅伺服器 | ❌ 客戶端瀏覽器 | ✅ 僅伺服器 |
| 離線支援 | ❌ 需要連線 | ✅ 可離線運行 | 🟡 部分支援 |
| 即時互動 (SignalR) | ✅ 低延遲 | ❌ 需要自行實作 | 🟡 複雜 |
| 檔案系統存取 | ✅ 直接在伺服器 | ❌ 需透過 API | ✅ 直接在伺服器 |
| 部署複雜度 | ⭐ 簡單 (單一站台) | ⭐ 簡單 (靜態檔案) | ⚠️ 需同時部署兩種模式 |
| 認證/授權 | ✅ 標準 ASP.NET Core | 🟡 需額外處理 | 🟡 複雜 |
| Docker 容器化 | ✅ 輕量 (單一映象) | ✅ 輕量 (靜態檔案) | ❌ 兩種映象或複雜設定 |

### 1.2 決定：✅ Blazor Server

**選擇理由：**

1. **檔案作業需要伺服器端存取**：檔案移動、刪除、建立等作業必須在具備檔案系統權限的伺服器端執行，Blazor Server 天生支援直接呼叫 .NET I/O API，無需透過額外的 WebAssembly interop。

2. **不需要離線能力**：FileManager 是內部工具，正常情況下會持續連線，離線作業並非必要需求。

3. **初始載入較快**：Blazor Server 無需下載 .NET WASM  runtime，使用者首次開啟即可快速互動，體驗更流暢。

4. **部署與維護簡單**：單一 ASP.NET Core 站台即可運作，容器化後僅需一個 Docker 映象，維運負擔最低。

5. **.NET 8 Blazor Server 成熟穩定**：SignalR 連線管理、檔案上傳streaming、元件狀態管理等議題在 .NET 8 已有完整支援。

6. **符合 Greenfield 重建策略**：功能需求明確，無需 Blazor United 的動態切換複雜度。

> **未來若有離線需求**，可考慮重構為 Blazor United，但初期 Phase 1-3 以 Blazor Server 為目標。

### 3.3 Debug Header 整合

所有 API 回應皆包含 Debug Header。

#### 請求端（Client → Server）

| Header | 說明 | 範例 |
|--------|------|------|
| `X-Trace-ID` | 外部追蹤 ID（可選，外部系統產生）| `P002-API-Browse-20260405-000001` |
| `X-Testing-Case` | 測試案例 ID（測試/QA 填寫）| `TC-FM001-001` |
| `X-QA-Debug-Info` | QA 除錯額外資訊（JSON，Base64 編碼）| `{"env":"staging","user":"tester01"}` |

#### 回應端（Server → Client）

| Header | 說明 | 範例 |
|--------|------|------|
| `X-Trace-ID` | 本次請求的 Trace ID | `P002-API-Browse-20260405-000001` |
| `X-Request-Duration-Ms` | 請求處理時間（毫秒）| `42` |
| `X-Server-Time` | 伺服器時間（ISO 8601）| `2026-04-05T22:00:00.000Z` |
| `X-QA-Debug-Info` | 若请求携带则回传（echo）| 同上 |

#### Debug Header 中介層實作

```csharp
// DebugTraceMiddleware.cs
public class DebugTraceMiddleware
{
    private readonly RequestDelegate _next;
    private readonly ILogger<DebugTraceMiddleware> _logger;

    public DebugTraceMiddleware(RequestDelegate next, ILogger<DebugTraceMiddleware> logger)
    {
        _next = next;
        _logger = logger;
    }

    public async Task InvokeAsync(HttpContext context)
    {
        // 取得或產生 Trace ID
        var traceId = context.Request.Headers["X-Trace-ID"].FirstOrDefault()
            ?? GenerateTraceId("API"); // 格式：P002-{維度}-{功能}-{YYYYMMDD}-{序號}

        // 計時
        var stopwatch = Stopwatch.StartNew();

        // 注入到 HttpContext.Items 供後續使用
        context.Items["TraceId"] = traceId;

        // 新增回應 Header
        context.Response.OnStarting(() =>
        {
            context.Response.Headers["X-Trace-ID"] = traceId;
            context.Response.Headers["X-Request-Duration-Ms"] = stopwatch.ElapsedMilliseconds.ToString();
            context.Response.Headers["X-Server-Time"] = DateTime.UtcNow.ToString("o");

            // 若请求携带 QA Debug，則 echo 回傳
            if (context.Request.Headers.ContainsKey("X-QA-Debug-Info"))
            {
                context.Response.Headers["X-QA-Debug-Info"] =
                    context.Request.Headers["X-QA-Debug-Info"].ToString();
            }

            return Task.CompletedTask;
        });

        // 結構化 Log
        using (_logger.BeginScope(new Dictionary<string, object>
        {
            ["TraceId"] = traceId,
            ["Method"] = context.Request.Method,
            ["Path"] = context.Request.Path.ToString()
        }))
        {
            await _next(context);
        }
    }
}
```

## 5. Debug Trace 設計

### 5.1 Trace ID 格式

```
P{專案}-{維度}-{功能}-{YYYYMMDD}-{序號}
```

**格式說明：**

| 欄位 | 說明 | 範例 |
|------|------|------|
| `P002` | 專案代號（P002=FileManager）| `P002` |
| `{維度}` | 作業維度 | `API`, `FILE`, `UPLOAD`, `SYS` |
| `{功能}` | 具體功能 | `Browse`, `Rename`, `Download`, `Search` |
| `{YYYYMMDD}` | 日期（UTC）| `20260405` |
| `{序號}` | 每日流水號（6位數）| `000001` |

**維度分類：**

| 維度 | 適用場景 |
|------|----------|
| `API` | 所有 HTTP API 請求 |
| `FILE` | 檔案讀寫操作 |
| `UPLOAD` | 檔案上傳作業 |
| `FOLDER` | 資料夾作業（建立/刪除/移動）|
| `SYS` | 系統層級作業（啟動、設定讀取）|

**範例 Trace ID：**

| 場景 | Trace ID |
|------|----------|
| API 瀏覽 `/api/files` | `P002-API-Browse-20260405-000001` |
| 檔案重新命名 | `P002-FILE-Rename-20260405-000002` |
| 多檔上傳 | `P002-UPLOAD-Multi-20260405-000003` |
| 資料夾建立 | `P002-FOLDER-Create-20260405-000004` |
| 系統啟動 | `P002-SYS-Startup-20260405-000001` |

### 5.2 Trace ID 產生器

```csharp
public interface ITraceIdGenerator
{
    string Generate(string dimension, string function);
}

public class TraceIdGenerator : ITraceIdGenerator
{
    private static int _counter = 0;
    private static readonly object _lock = new();
    private static string _lastDate = string.Empty;

    public string Generate(string dimension, string function)
    {
        var today = DateTime.UtcNow.ToString("yyyyMMdd");

        lock (_lock)
        {
            if (_lastDate != today)
            {
                _lastDate = today;
                _counter = 0;
            }
            _counter++;
            return $"P002-{dimension}-{function}-{today}-{_counter:D6}";
        }
    }
}
```

## 6. 安全性設計摘要

| 威脅 | 防護機制 |
|------|----------|
| 路徑遍歷 (`../`) | 所有路徑必須在 `RootFolder` 內；使用 `Path.GetFullPath()` + `StartsWith()` 驗證 |
| 任意檔案下載 | 路徑必須在 `RootFolder` 內；禁止 `..` |
| 檔案類型上傳 | 依 `AllowedFileExtensions` 驗證副檔名 |
| 上傳大小上限 | `MaxRequestLength` 限制；大檔使用串流上傳 |
| 路徑遍歷錯誤 | 統一在 `FileService` 處理路徑驗證，回傳 `PATH_TRAVERSAL_DETECTED` |
| 作業權限繞過 | Controller 層先檢查 `PermissionOptions` 各項設定 |
