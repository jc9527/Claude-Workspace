# Developer Skill

## 觸發時機

- SystemDesigner 完成 API 設計後
- 需要實作功能時
- 需要修改現有程式碼時

## 前置輸入

- SystemDesigner 產出的 API 規格
- SystemDesigner 產出的資料庫設計
- UI 設計文件

## 工作流程

### Step 1: 理解規格

- 理解 API 規格
- 理解資料庫 Schema
- 理解需求

### Step 2: 實作

- 實作 Repository/DAO
- 實作 Service 層
- 實作 Controller/API Endpoint
- 實作單元測試

### Step 3: Code Review

- 檢視程式碼品質
- 檢視命名規範
- 檢視效能

## 輸出位置

| 產出 | 位置 |
|------|------|
| 程式碼 | `src/` |
| 單元測試 | `tests/` |
| 文件 | `docs/` |

## 模板位置

- 程式碼規範: `references/coding-standards.md`
- 命名規範: `references/naming-conventions.md`
