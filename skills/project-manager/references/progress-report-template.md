## 完整範例

```markdown
# 專案進度報告

**專案名稱**：會員管理系統
**報告期間**：2026-04-01 ~ 2026-04-05
**報告編號**：PR-004
**報告日期**：2026-04-05
**報告人**：PM

---

## 1. 專案概覽

### 1.1 專案目標
建立會員管理系統，提供會員註冊、登入、資料管理功能。

### 1.2 目前進度
70%

### 1.3 里程碑狀態

| 里程碑 | 預定日期 | 實際日期 | 狀態 | 備註 |
|--------|----------|----------|------|------|
| 需求分析完成 | 2026-04-01 | 2026-04-01 | ✅ 完成 | |
| 系統設計完成 | 2026-04-03 | 2026-04-04 | ⚠️ 延遲 | 延遲 1 天 |
| 開發完成 | 2026-04-10 | - | 🔄 進行中 | |
| 測試完成 | 2026-04-15 | - | ⏳ 待開始 | |
| 上線 | 2026-04-20 | - | ⏳ 待開始 | |

---

## 2. 本期進度

### 2.1 已完成工作

| # | 工作項目 | 負責人 | 完成日期 | 說明 |
|---|----------|--------|----------|------|
| 1 | 會員登入 API | Dev-1 | 2026-04-03 | |
| 2 | 會員註冊 API | Dev-2 | 2026-04-04 | |
| 3 | 會員資料修改 API | Dev-1 | 2026-04-05 | |

### 2.2 進行中工作

| # | 工作項目 | 負責人 | 預定完成 | 進度 | 說明 |
|---|----------|--------|----------|------|------|
| 1 | 會員列表 API | Dev-2 | 2026-04-06 | 70% | 預計如期完成 |
| 2 | 密碼重設功能 | Dev-1 | 2026-04-07 | 50% | 需要第三方串接 |

---

## 3. 風險追蹤

| # | 風險 | 影響 | 機率 | 對策 | 狀態 |
|---|------|------|------|------|------|
| 1 | 密碼重設串接延遲 | 中 | 高 | 先行實作本地驗證 | 🔴 監控中 |
```

## 進度報告頻率建議

| 專案階段 | 建議頻率 |
|----------|----------|
| 專案啟動 | 每週 2 次 |
| 開發階段 | 每週 1 次 |
| 測試階段 | 每週 2 次 |
| 上線前 | 每日 |

---

## 進度報告檢查清單

- [ ] 所有里程碑狀態已更新
- [ ] 風險和問題都有記錄
- [ ] 下期工作有規劃
- [ ] 數據準確無誤
- [ ] 有具體的交付物說明
