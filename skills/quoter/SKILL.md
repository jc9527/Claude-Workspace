# Quoter Skill

## 觸發時機

- User Stories 完成後
- System Architecture 確認後
- 需要估算成本和時程時
- 需要產出正式報價單時

## 前置輸入

- User Stories（數量、複雜度）
- System Architecture（技術堆疊、應用程式數量）
- 人力單價（預設或客戶提供）
- 產品/授權費用

## 工作流程

### Step 1: 理解需求範圍

- 確認 Application 數量
- 確認功能複雜度
- 識別技術風險

### Step 2: 人力估算

- 估算每個 User Story 的工作量
- 計算總人天
- 套用單價計算成本

### Step 3: 產品成本估算

- 第三方服務費用
- 基礎設施費用
- 授權費用

### Step 4: 時程估算

- 根據人月/人天估算
- 考量並行開發
- 加入 Buffer

### Step 5: 產出報價單

- 格式化報價文件
- 分項列出成本明細
- 提供選項（不同方案）

## 專家知識

### 常見估算模型

| 模型 | 說明 |
|------|------|
| **經驗估算** | 根據類似專案經驗 |
| **功能點數法** | FP × 生產力 = 人天 |
| **類比估算** | 與過去專案比較 |
| **三點估算** | (樂觀 + 4×可能 + 悲觀) / 6 |

### 成本組成

| 類別 | 說明 |
|------|------|
| 人力成本 | 開發、測試、PM |
| 軟體成本 | 授權、第三方服務 |
| 硬體成本 | 伺服器、設備 |
| 維運成本 | 第一年免費 or 收費 |
| 風險溢價 | 通常加 10-20% |
