## 風險溢價

| 風險類型 | 溢價比例 | 說明 |
|----------|----------|------|
| 需求不確定 | 10-20% | 客戶需求模糊 |
| 技術風險 | 10-15% | 新技術、不熟悉領域 |
| 時程緊迫 | 10-20% | 高於一般工期 |
| 人力流動 | 5-10% | 替換人員成本 |
| 整合複雜 | 10-15% | 第三方系統串接 |

**建議**：風險溢價一般抓 10-20%

## User Story 估算方法

### 快速估算（經驗法）

| 複雜度 | 人天 | 範例 |
|---------|------|------|
| 簡單 | 0.5-1 | 純 CRUD |
| 中等 | 1-2 | 有驗證、簡單商業邏輯 |
| 複雜 | 2-5 | 多表關聯、計算邏輯 |
| 極複雜 | 5+ | 全新系統、演算法 |

### 對照表

| 動作 | 預估人天 |
|------|----------|
| 新增一筆資料 | 0.5 |
| 編輯一筆資料（含驗證） | 1 |
| 刪除一筆資料（含確認） | 0.5 |
| 查詢列表（分頁、篩選） | 1 |
| 詳細檢視 | 0.5 |
| 單一 API Endpoint | 0.5-1 |
| 資料庫單一 Table | 0.5 |
| 簡單報表 | 1-2 |
| 複雜報表（含圖表） | 3-5 |

## 估算調整因素

### 乘數

| 因素 | 乘數 |
|------|------|
| 全新技術（非團隊熟悉） | 1.5-2.0 |
| 高品質要求（金融、醫療） | 1.3-1.5 |
| 緊迫時程 | 1.2-1.5 |
| 國籍/遠端團隊 | 0.8-1.2 |
| 有類似經驗 | 0.7-0.9 |
