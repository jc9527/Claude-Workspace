# User Story 模板

## 基本格式

```markdown
## User Story #[N]

**作為** [角色]
**我希望** [功能]
**以便** [價值/利益]

### 驗收標準

| 情境 | 預期結果 |
|------|----------|
| Given [前提條件] | |
| When [操作] | |
| Then [預期輸出] | |

### 補充說明

- **优先级**：Must Have / Should Have / Could Have / Won't Have
- **估算**：?[ 小時/天]
- **依賴**： 無 / [依賴的 Story 或系統]
- **風險**： [識別的風險]
```

---

## 完整範例

```markdown
## User Story #1

**作為** 線上購物網站的會員
**我希望** 在結帳頁面查看訂單摘要
**以便** 確認商品、數量、價格無誤後再完成付款

### 驗收標準

| 情境 | 預期結果 |
|------|----------|
| Given 會員已登入並擁有購物車內容 | |
| When 會員進入結帳頁面 | |
| Then 顯示購物車中所有商品的名稱、單價、數量 | |

| 情境 | 預期結果 |
|------|----------|
| Given 會員已登入並擁有購物車內容 | |
| When 會員進入結帳頁面 | |
| Then 顯示總金額（含運費） | |

| 情境 | 預期結果 |
|------|----------|
| Given 會員在結帳頁面 | |
| When 會員點擊「編輯」按鈕 | |
| Then 返回購物車頁面進行修改 | |

### 補充說明

- **優先級**：Must Have
- **估算**：2 天
- **依賴**：購物車系統（Story #2）
- **風險**：無

## Epic vs Story

### Epic（史詩級，用於大型功能）

```markdown
# Epic: 會員管理系統

作為 網站管理者
我希望 會員管理功能
以便 管理會員資料和權限

## 相關 Stories

- Story #1: 會員註冊
- Story #2: 會員登入
- Story #3: 會員資料修改
- Story #4: 會員權限管理
```

### Story（一般 User Story）

用於 1-3 天可完成的顆粒度。
```

---

## 模板使用說明

1. 每次新建 Story 複製此模板
2. 確保角色、功能、價值三要素完整
3. 驗收標準要具體且可測試
4. 優先使用 Given-When-Then 格式
