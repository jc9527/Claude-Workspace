# Use Case（使用案例）模板

---

## 基本格式

```markdown
# Use Case: [功能名稱]

## 基本資訊

| 欄位 | 內容 |
|------|------|
| **使用案例編號** | UC-[001] |
| **名稱** | [功能名稱] |
| **主要参与者** | [誰會使用這個功能] |
| **目標** | [使用者想要達成什麼] |
| **前置條件** | [在開始之前必須滿足的條件] |
| **後置條件** | [成功完成後系統的狀態] |

---

## 主要流程

| 步驟 | 參與者 | 動作 |
|------|--------|------|
| 1 | 會員 | 會員點擊「註冊」按鈕 |
| 2 | 系統 | 系統顯示註冊表單 |
| 3 | 會員 | 會員填寫 email、密碼、確認密碼 |
| 4 | 會員 | 會員點擊「送出」 |
| 5 | 系統 | 系統驗證資料格式 |
| 6 | 系統 | 系統檢查 email 是否已被使用 |
| 7 | 系統 | 系統建立新會員帳戶 |
| 8 | 系統 | 系統發送驗證 email |
| 9 | 系統 | 系統顯示「註冊成功」訊息 |

---

## 替代流程

### 替代流程 1：Email 已被使用

| 步驟 | 條件 | 動作 |
|------|------|------|
| 6a | 如果 email 已存在 | 系統顯示「此 email 已被註冊」 |
| 6a.1 | 會員 | 會員輸入不同的 email |
| 6a.2 | | 回到步驟 5 |

### 替代流程 2：資料驗證失敗

| 步驟 | 條件 | 動作 |
|------|------|------|
| 5a | 如果格式不符 | 系統顯示「格式錯誤：...」 |
| 5a.1 | 會員 | 會員修正錯誤欄位 |
| 5a.2 | | 回到步驟 4 |

---

## 例外流程

### 例外流程 1：系統錯誤

| 步驟 | 條件 | 動作 |
|------|------|------|
| 7a | 如果資料庫連線失敗 | 系統顯示「系統錯誤，請稍後再試」 |
| 7a.1 | | 記錄錯誤 Log，通知管理員 |

---

## 補充說明

- **頻率**：每週約 100 次
- **假設**：會員有有效的 email 地址
- **備註**：[其他重要資訊]
```

## 完整範例：會員登入

```markdown
# Use Case: 會員登入

## 基本資訊

| 欄位 | 內容 |
|------|------|
| **使用案例編號** | UC-002 |
| **名稱** | 會員登入 |
| **主要参与者** | 會員 |
| **目標** | 會員成功登入系統 |
| **前置條件** | 會員已完成註冊並擁有有效帳號 |
| **後置條件** | 會員成功進入系統首頁，session 已建立 |

---

## 主要流程

| 步驟 | 參與者 | 動作 |
|------|--------|------|
| 1 | 會員 | 會員在首頁點擊「登入」 |
| 2 | 系統 | 系統顯示登入表單（email、密碼） |
| 3 | 會員 | 會員輸入 email 和密碼 |
| 4 | 會員 | 會員點擊「登入」按鈕 |
| 5 | 系統 | 系統驗證 email 格式 |
| 6 | 系統 | 系統檢查帳號是否存在 |
| 7 | 系統 | 系統驗證密碼是否正確 |
| 8 | 系統 | 系統建立 session |
| 9 | 系統 | 系統重新導向至會員首頁 |

---

## 替代流程

### 替代流程 1：密碼錯誤

| 步驟 | 條件 | 動作 |
|------|------|------|
| 7a | 如果密碼錯誤 | 系統顯示「帳號或密碼錯誤」 |
| 7a.1 | 會員 | 會員重新輸入密碼 |
| 7a.2 | | 回到步驟 4 |
| 7a.3 | | （最多嘗試 5 次，見例外流程 2） |

---

## 例外流程

### 例外流程 1：帳號不存在

| 步驟 | 條件 | 動作 |
|------|------|------|
| 6a | 如果帳號不存在 | 系統顯示「帳號或密碼錯誤」 |

### 例外流程 2：登入嘗試次數超過限制

| 步驟 | 條件 | 動作 |
|------|------|------|
| 7b | 如果連續錯誤達 5 次 | 系統鎖定帳號 30 分鐘 |
| 7b.1 | | 顯示「帳號已鎖定，請 30 分鐘後再試」 |
| 7b.2 | | 發送通知 email 給會員 |

---

## 驗收標準

| 情境 | 預期結果 |
|------|----------|
| 正確帳密 | 成功登入，跳轉至首頁 |
| 錯誤密碼 | 顯示錯誤訊息，留在登入頁 |
| 帳號不存在 | 顯示錯誤訊息，留在登入頁 |
| 連續 5 次錯誤 | 鎖定帳號 30 分鐘 |
| 空白欄位 | 提示「必填欄位」 |
```

## Use Case 寫作檢查清單

- [ ] 參與者（Actor）是否正確？
- [ ] 前置/後置條件是否完整？
- [ ] 主要流程是否清楚？
- [ ] 替代流程是否完整？（各種分支）
- [ ] 例外處理是否有說明？
- [ ] 步驟是否有順序性？
- [ ] 每個步驟是否由 Actor 或系統執行的動作？
- [ ] 系統回應是否清楚？
