## 完整範例：會員登入

```markdown
# 測試案例：會員登入

**測試案例編號**：TC-001
**功能**：會員登入
**測試類型**：功能測試

---

## 基本資訊

| 欄位 | 內容 |
|------|------|
| **建立日期** | 2026-04-05 |
| **測試人員** | QA Engineer |
| **執行日期** | 2026-04-05 |
| **測試結果** | Pass |

---

## 前置條件

- 會員已完成註冊，帳號 `test@example.com`，密碼 `Pass123`

---

## 測試資料

| 欄位 | 值 |
|------|------|
| Email | test@example.com |
| Password | Pass123 |

---

## 測試步驟

| 步驟 | 動作 | 預期結果 |
|------|------|----------|
| 1 | 開啟登入頁面 | 顯示登入表單 |
| 2 | 輸入正確 Email | 欄位顯示輸入值 |
| 3 | 輸入正確密碼 | 密碼顯示為 *** |
| 4 | 點擊「登入」按鈕 | 跳轉至會員首頁 |
| 5 | 顯示「登入成功」訊息 | 顯示成功提示 |

---

## 測試結果

| 項目 | 結果 |
|------|------|
| 執行結果 | Pass |
| 發現問題數 | 0 |

---

## 問題記錄

（無）
```

## 常見測試類型

| 類型 | 說明 |
|------|------|
| **功能測試** | 驗證功能是否正常運作 |
| **整合測試** | 驗證模組之間的整合 |
| **系統測試** | 驗證整個系統 |
| **效能測試** | 驗證效能指標 |
| **安全性測試** | 驗證安全性 |

---

## 嚴重度分類

| 等級 | 說明 |
|------|------|
| **Critical** | 系統無法運作 |
| **High** | 主要功能失效 |
| **Medium** | 功能有問題但不影響主要流程 |
| **Low** | 介面或顯示問題 |

---

## 測試案例寫作檢查清單

- [ ] 每個步驟都有預期結果
- [ ] 覆盖正向和負向測試
- [ ] 测试资料明确
- [ ] 前置條件清楚
- [ ] 测试可重复执行
