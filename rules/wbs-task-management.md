# WBS 任務管理規範 (SOP)

## 目的

建立標準化的任務分派流程，確保強哥能掌控每個工作項目的規劃與執行。

---

## 檔案位置

```
Claude-Workspace/
├── rules/wbs-task-management.md   # 本規範
└── projects/<專案名>/
    ├── plan.md                   # 專案需求
    ├── wbs.md                    # 工作分解結構
    ├── prompts/                  # 提示詞
    │   ├── P001-001.md
    │   └── P001-002.md
    ├── outputs/                  # 產出檔案
    └── flows/                    # ClawFlow 狀態
```

---

## 核心流程

```
1️⃣ 接收需求
2️⃣ 產生 WBS
3️⃣ 你確認 WBS
4️⃣ 產生提示詞
5️⃣ 你確認提示詞
6️⃣ 派工執行
7️⃣ 監控回報
```

---

## 詳細步驟

### 1️⃣ 接收需求

- 強哥輸入需求描述
- 我記錄到 `projects/<專案名>/plan.md`
- 等待強哥確認需求

### 2️⃣ 產生 WBS

- 根據需求產生 WBS 表格
- 存到 `projects/<專案名>/wbs.md`

**WBS 格式：**

| ID | 摘要 | 相依性 | 建議 Agent | 狀態 |
|----|------|--------|------------|------|
| P001-001 | 市場分析 | None | BA | 📋 |
| P001-002 | 技術評估 | None | Architect | 📋 |
| P001-003 | 成本估算 | P001-001, P001-002 | Quoter | 📋 |

**狀態說明：**
- 📋 待規劃
- ✅ 已確認
- 🚀 已派工
- ⏸ 暫停
- 🛑 終止
- 🎉 完成

### 3️⃣ 你確認 WBS

- 強哥確認每個工作項
- 可修改、刪除、新增項目
- 確認後我才會繼續

### 4️⃣ 產生提示詞

- 對每個已確認的 WBS，產生具體提示詞
- 存到 `prompts/P001-001.md` 等

### 5️⃣ 你確認提示詞

- 派工前再次跟強哥確認
- 強哥說 OK 才執行
- 可以修改提示詞後再確認

### 6️⃣ 派工執行

- 使用 `sessions_spawn` 派給 sub-agent
- 獨立執行，不等待另一個 Agent
- 追蹤執行狀態

### 7️⃣ 監控回報

## 指令對照

| 強哥指令 | 我的動作 |
|----------|----------|
| 狀態 | 回報所有任務進度 |
| P001-001 進度 | 詳細報告這項任務 |
| 停止 P001-001 | 立即終止該任務 |
| 暫停 P001-001 | 標記暫停，等你恢復 |
| 恢復 P001-001 | 繼續執行被暫停的任務 |
| 全部停止 | 終止所有進行中的任務 |

---

## 通知格式

| 情境 | 格式 |
|------|------|
| WBS 產生 | 📋 WBS 已完成，等待你確認 |
| 派工確認 | 🚀 P001-001 準備派工，等待你確認 |
| 執行中 | ⏳ P001-001 執行中（BA 分析中...） |
| 完成 | ✅ P001-001 完成：結果摘要 |
| 需要確認 | ⚠️ P001-003 需要你確認：問題描述 |
| 暫停 | ⏸ P001-001 已暫停 |
| 中斷 | 🛑 P001-001 已终止 |
| 全部完成 | 🎉 專案完成！ |

---

## 中斷確認機制

當 Agent 需要確認時：
- 我轉告強哥：問題 + 選項
- 強哥回覆指示
- 我傳給 Agent 繼續執行

**範例：**
```
⚠️ 需要確認
Agent: BA
Task: P001-001 市場分析
問題：找到了兩種市場趨勢，要選哪一個？
選項：① A 趨勢 ② B 趨勢 ③ 兩者都分析
```

---

## 原則

1. **一次一專案** - 同時只有一個活躍專案
2. **派工前確認** - 每個任務執行前都會再次確認
3. **即時回報** - 每件完成都通知強哥
4. **可中斷** - 強哥可隨時停止或暫停任務
5. **獨立執行** - Agent 之间不互相等待
