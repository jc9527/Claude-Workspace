# Idea 處理流程規範

## 原則

**任何 Idea 討論後，絕對不可直接執行。必須先提供建議 Plan 與複雜度評估。**

---

## 流程

1. **傾聽理解** — 了解用戶想要解決的問題或需求
2. **分析評估** — 評估技術可行性和複雜度
3. **提供 Plan** — 產出建議的實作計劃
4. **等待確認** — 用戶確認後才可執行

---

## Plan 內容要求

每個 Idea 的回應必須包含：

| 項目 | 說明 |
|------|------|
| **目標** | 用戶想要達成什麼 |
| **步驟** | 具體實作順序（編號） |
| **複雜度** | Low / Medium / High（說明理由） |
| **預估時間** | 大概需要多久 |
| **所需工具** | 需要哪些工具或技術 |
| **風險** | 可能遇到的問題 |

---

## 複雜度定義

| 等級 | 定義 |
|------|------|
| **Low** | 1–2 個 Script 或簡單設定，1 小時內完成 |
| **Medium** | 多個步驟、需要整合多項工具，1 天內完成 |
| **High** | 涉及架構設計、多系統整合、超過 1 天 |

---

## 範例

### ❌ 錯誤（直接執行）

> 用戶：「我想自動化這個流程。」
> AI：（直接開始寫腳本並執行）

### ✅ 正確（先提供 Plan）

> 用戶：「我想自動化這個流程。」
> AI：「好的，我建議這樣做：
> 
> **目標**：自動化 XXX 流程
> 
> **Plan**：
> 1. 建立收集資料的 Script
> 2. 建立分析邏輯的 Prompt
> 3. 設定排程
> 
> **複雜度**：Medium
> **原因**：需要串接多個工具
> 
> **預估時間**：2–3 小時
> 
> 請確認是否按照這個方向進行？"
